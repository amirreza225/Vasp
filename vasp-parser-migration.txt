New v2 DSL — Concrete Syntax Reference
vasp
entity Todo {
  id:        Int      @id
  title:     String {
    label:       "Task Title"
    placeholder: "Enter a task name…"
    description: "The primary text of the todo item"
    validate {
      minLength: 3
      maxLength: 120
      required:  true
    }
  }
  priority: Enum(low, medium, high) {
    label:   "Priority"
    default: "medium"
  }
  done: Boolean {
    label:   "Completed"
    default: false
  }
  createdAt: DateTime @default(now)
}

crud Todo {
  entity:     Todo
  operations: [list, create, update, delete]

  list {
    paginate:   true
    sortable:   [title, createdAt, priority]
    filterable: [done, priority]
    search:     [title]
    columns {
      title {
        label:     "Task"
        width:     "40%"
        sortable:  true
      }
      priority {
        label:     "Priority"
        width:     "120px"
        filterable: true
      }
      done {
        label: "Done"
        width: "80px"
      }
    }
  }

  form {
    layout: "2-column"       // or "1-column" | "tabs" | "steps"

    sections {
      basics {
        label:  "Basic Info"
        fields: [title, priority]
      }
      status {
        label:  "Status"
        fields: [done]
      }
    }

    // OR, for wizard-style forms:
    steps {
      step1 {
        label:  "Details"
        fields: [title]
      }
      step2 {
        label:  "Settings"
        fields: [priority, done]
      }
    }
  }

  permissions {
    list:   [admin, user]
    create: [admin, user]
    update: [admin]
    delete: [admin]
  }
}

export interface FieldValidateConfig {
  required?:   boolean
  minLength?:  number
  maxLength?:  number
  min?:        number
  max?:        number
  pattern?:    string   // regex string
  custom?:     string   // import path to a validator fn
}

export interface EntityFieldConfig {
  label?:       string
  placeholder?: string
  description?: string
  default?:     string | number | boolean
  validate?:    FieldValidateConfig
}

// Extend existing FieldNode
export interface FieldNode extends BaseNode {
  // ... existing fields ...
  config?: EntityFieldConfig     // ← NEW: present when field has { ... } block
}

// --- CRUD nested configs ---
export interface CrudColumnConfig {
  label?:      string
  width?:      string
  sortable?:   boolean
  filterable?: boolean
  hidden?:     boolean
}

export interface CrudListConfig {
  paginate?:   boolean
  sortable?:   string[]
  filterable?: string[]
  search?:     string[]
  columns?:    Record<string, CrudColumnConfig>
}

export interface CrudFormSection {
  label?:  string
  fields:  string[]
}

export interface CrudFormConfig {
  layout?:   '1-column' | '2-column' | 'tabs' | 'steps'
  sections?: Record<string, CrudFormSection>
  steps?:    Record<string, CrudFormSection>
}

// Extend existing CrudNode
export interface CrudNode extends BaseNode {
  // ... existing fields ...
  listConfig?:   CrudListConfig   // ← NEW
  formConfig?:   CrudFormConfig   // ← NEW
}

Key Chevrotain grammar rules
Code
VaspFile       → BlockDecl*
BlockDecl      → AppBlock | EntityBlock | CrudBlock | AuthBlock | RouteBlock | ...
EntityBlock    → KW_ENTITY Identifier LBRACE FieldDecl* RBRACE
FieldDecl      → Identifier COLON FieldType ModifierList? (LBRACE FieldConfig RBRACE)?
FieldConfig    → (KW_LABEL COLON StringLiteral | KW_PLACEHOLDER COLON StringLiteral | ValidateBlock | ...)*
CrudBlock      → KW_CRUD Identifier LBRACE CrudProp* CrudListBlock? CrudFormBlock? PermissionsBlock? RBRACE
CrudListBlock  → KW_LIST LBRACE ListProp* ColumnsBlock? RBRACE
ColumnsBlock   → KW_COLUMNS LBRACE ColumnDecl* RBRACE
ColumnDecl     → Identifier LBRACE ColumnProp* RBRACE
...
Error recovery
Chevrotain's built-in IN_RECOVERY_MODE + RECOVER_INSERT/RECOVER_DELETE ensures partial documents still produce a partial CST — diagnostics show only the actual error, not "everything after line 5 is broken".

features/diagnostics.ts
Parse with Chevrotain → collect parse errors → convert to Diagnostic[]
Run SemanticValidator-equivalent checks on the CST AST
Debounce: 300ms after last keystroke
features/completions.ts
Context states (detected from cursor position):

Context Completions
Top-level (no block open) All 20 block type keywords + snippets
Inside job {}, after executor: PgBoss / BullMQ / RedisStreams / RabbitMQ / Kafka — each with auto-insert snippet
Inside crud {}, start of line list, form, permissions sub-block keywords
Inside crud.list {} paginate, sortable, filterable, search, columns
Inside crud.form {} layout, sections, steps
Inside entity {}, after field type All @modifier names
Inside auth {}, after methods: usernameAndPassword, google, github
After entity: in crud {} All declared entity names from workspace
After to: in route {} All declared page names from workspace
After provider: in storage {} local, s3, r2, gcs
After provider: in email {} resend, sendgrid, smtp
After provider: in cache {} memory, redis, valkey
After exporter: in observability {} console, otlp, prometheus
After errorTracking: none, sentry, datadog
